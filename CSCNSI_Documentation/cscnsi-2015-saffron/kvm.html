KVM SR-IOV and VMs

Install KVM
Install the packages
~ yum -y install kvm qemu-kvm libvirt python-virtinst bridge-utils

Make sure the modules are loaded
~ lsmod | grep kvm

Start the services
~ service libvirtd start
~ service messagebus start

- opensm and rdma are required but must be start after 

Enable them in the chkconfig
~ chkconfig libvirtd on
~ chkconfig messagebus on

Enable SR-IOV
Add the line iommu=1 to the end of the kernel line so that SR-IOV is enabled.
The /etc/grub.conf entry should look as below
#
# Note that you do not have to rerun grub after making changes to this file
# NOTICE:  You have a /boot partition.  This means that
#          all kernel and initrd paths are relative to /boot/, eg.
#          root (hd0,0)
#          kernel /vmlinuz-version ro root=/dev/mapper/system-lv_root
#          initrd /initrd-[generic-]version.img
#boot=/dev/sda
default=0
timeout=5
splashimage=(hd0,0)/grub/splash.xpm.gz
hiddenmenu
title CentOS 6 (2.6.32-504.23.4.el6.centos.plus.x86_64)
        root (hd0,0)
        kernel /vmlinuz-2.6.32-504.23.4.el6.centos.plus.x86_64 ro root=/dev/mapper/system-lv_root rd_LVM_LV=system/lv_root rd_NO_LUKS LANG=en_US.UTF-8 rd_NO_MD  KEYTABLE=us SYSFONT=latarcyrheb-sun16 rd_LVM_LV=system/lv_swap crashkernel=auto rhgb quiet rd_NO_DM rhgb quiet iommu=1 intel_iommu=on
        initrd /initramfs-2.6.32-504.23.4.el6.centos.plus.x86_64.img

Add a line to /etc/modprobe.d/modprobe.conf (or create something new like mlx4_core.conf) to specify the number of virtual functions you would like
options mlx4_core num_vfs=X probe_vf=Y
Page 27 of http://www.mellanox.com/related-docs/prod_software/Mellanox_EN_for_Linux_User_Manual_v2_0-3_0_0.pdf details what these parameters do

reboot the machine

Check that they were enabled
~ lspci | grep Mellanox

Create a VM
Create the directory
~ mkdir -p /home/kvm/images/

Create a network bridge
~ vi /etc/sysconfig/network-scripts/ifcfg-br0
DEVICE="br0"
BOOTPROTO="dhcp"
HWADDR="38:EA:A7:8B:00:A4"
IPV6INIT="no"
MTU="1500"
NM_CONTROLLED="yes"
ONBOOT="yes"
TYPE="Bridge"
UUID="157ae3d8-048b-4e3c-8a9f-93762bc5e9bb"

[root@localhost ~]# cat /etc/sysconfig/network-scripts/ifcfg-eth0
DEVICE="eth0"
BOOTPROTO="static"
HWADDR="38:EA:A7:8B:00:A4"
IPV6INIT="no"
MTU="1500"
NM_CONTROLLED="yes"
ONBOOT="yes"
TYPE="Ethernet"
UUID="157ae3d8-048b-4e3c-8a9f-93762bc5e9bb"
BRIDGE=br0
IPADDR="192.168.0.10"
NETMASK="255.255.254.0"
GATEWAY="192.168.0.1"

Kickstart file
~ vi kvmks.cfg
auth  --useshadow  --enablemd5
bootloader --location=mbr
zerombr
clearpart --all --initlabel
text
firstboot --disable
keyboard us
network --device eth0 --bootproto dhcp --onboot yes --noipv6
url --url http://repo.newmexicoconsortium.org/centos/6/os/x86_64
rootpw password
lang en_US
logging --level=info

reboot
selinux --disable
skipx
timezone  --utc MST
install
part / --bytes-per-inode=4096 --fstype="ext3" --grow --size=1
part swap --recommended
%packages
@core

%post
# repos
rm /etc/yum.repos.d/*
curl saffron.com/cfg/CentOS-NMC.repo > /etc/yum.repos.d/CentOS-BaseSaffron.repo
curl saffron.com/cfg/lustre.repo > /etc/yum.repos.d/lustre.repo

# ib interface
curl saffron.com/cfg/ifcfg-ib0 > /etc/sysconfig/network-scripts/ifcfg-ib0

# installs
yum -y install openssh-server ntp nfs-utils
yum -y install lustre-testsp
yum -y groupinstall "Infiniband Support"
yum -y --setopt=group_package_types=optional groupinstall "Infiniband Support"
yum -y install environment-modules openmpi openmpi-devel

# ntp config
curl repo.newmexicoconsortium.org/cfg/ntp.conf > /etc/ntp.conf
# fstab to mount /home <- no this screws stuff up
# curl repo.newmexicoconsortium.org/cfg/fstab > /etc/fstab
echo 'nfs.saffron.com:/home/  /home                   nfs     defaults        0 0' >> /etc/fstab
# lustre
echo "modprobe lnet" >> /etc/rc.local
echo "options lnet networks=o2ib(ib0)" > /etc/modprobe.d/lustre.conf

# chkconfig
chkconfig rdma on

Create VM
~ virt-install -n test0 -r 4096 --vcpus=2 -v --network bridge=br0,mac=38:EA:A7:8A:00:A5 -l http://repo.newmexicoconsortium.org/centos/6/os/x86_64 -x "ks=http://192.168.0.1/cfg/kvmks.cfg text console=tty0 console=ttyS0,115200" --disk=/home/kvm/images/test0,size=50 --nographics --noautoconsole

Create an xml file for the 
~ vi pci.xml
<hostdev mode='subsystem' type='pci' managed='yes'>
  <source>
     <address domain='0' bus='33' slot='0' function='1'/>
  </source>
</hostdev>

The VM must be started for the attach-device command
~ virsh start test0

Attache the device
~ virsh attach-device test0 pci.xml --persistent

Verify that it worked in the host
~ lspci
...
00:07.0 Network controller: Mellanox Technologies MT27500/MT27520 Family [ConnectX-3/ConnectX-3 Pro Virtual Function]


Clone a VM
~ virsh suspend test0
~ virt-clone -o test0 --name test1 --file /home/kvm/images/test1

Resizing a VM
Find the existing partitions and sizes on a disk
~ virt-filesystems --long --parts --blkdevs -h -a /home/kvm/images/test1
Name       Type       MBR  Size  Parent
/dev/sda1  partition  82   819M  /dev/sda
/dev/sda2  partition  83   13G   /dev/sda
/dev/sda   device     -    14G   -

Create a file the size you want to end up with
~ truncate -s 15G /home/outfile

Resize the disk by expanding the current partition, otherwise it will just make a blank partition at the end to take up the remaining space.
~ virt-resize --expand /dev/sda2 /home/kvm/images/test1 /home/outdisk

Back up the current file and then move the outfile to the correct location
~ mv /home/kvm/images/test1 /home/kvm/images/test1.bak
~ mv /home/outdisk /home/kvm/images/test1

Start the VM and verify it worked in the VM
~ virsh start test1
