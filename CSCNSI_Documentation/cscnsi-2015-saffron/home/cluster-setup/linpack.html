linpack

Introduction
Linpack is a tool used to benchmark (in terms of FLOPs) by computing the solution to a random linear system Ax=b.

Installation
We are doing a parallel benchmark so MPI and Infiniband should be setup (see appropriate page)

Grab HPL (high performance linpack). There may be a newer version (check http://www.netlib.org/benchmark/hpl/).
~ wget http://www.netlib.org/benchmark/hpl/hpl-2.1.tar.gz

You will also need BLAS (or VSIPL)
~ yum -y install openblas.x86_64 openblas-devel.x86_64

Configuration
It's easiest if you build in ~/ so copy the untared directory there and rename it to hpl (this is because on of the make files references from ~/hpl explicitly...)
Grab the appropriate Makefile from ../hpl/setup (we used Make.Linux_PII_CBLAS_gm) and place it in ../hpl/
This file needs to be updated so that the libraries and paths are correct, below is a snippet of our file.
...
# ----------------------------------------------------------------------
# - Message Passing library (MPI) --------------------------------------
# ----------------------------------------------------------------------
# MPinc tells the C compiler where to find the Message Passing library
# header files, MPlib is defined to be the name of the library to be
# used. The variable MPdir is only used for defining MPinc and MPlib.
#
MPdir =
MPinc =
MPlib = /usr/lib64/openmpi/lib/libmpi.so
#
# ----------------------------------------------------------------------
# - Linear Algebra library (BLAS or VSIPL) -----------------------------
# ----------------------------------------------------------------------
# LAinc tells the C compiler where to find the Linear Algebra library
# header files, LAlib is defined to be the name of the library to be
# used. The variable LAdir is only used for defining LAinc and LAlib.
#
LAdir =
LAinc =
LAlib = /usr/lib64/libopenblas.so
...
We did not modify any other portion of our file (OpenMPI binarys were in PATH because the module was loaded.

After this build the source (from ../hpl/)
~ make arch=Linux_PII_CBLAS_gm
this should put a binary called xhpl somewhere so just find it.

We used (http://www.advancedclustering.com/act-kb/tune-hpl-dat-file/) to generate a base HPL.dat and modified it slightly.
Below is our best configuration (it can probably be better)
