InfiniBand/OpenMPI

Installation (all nodes)
# yum -y groupinstall "Infiniband Support"
# yum -y --setopt=group_package_types=optional groupinstall "Infiniband Support"
# yum -y install environment-modules openmpi openmpi-devel

Configuration (all nodes)
InfiniBand
Turn rdma on and enable at boot
# service rdma start
# chkconfig rdma on

You should be able to see all hosts on your IB network
# ibhosts

Ca : 0x0002c90300eca0d0 ports 1 "n10 mlx4_0"

Head
Set a static IP for the ib interface on the head node
/etc/syslog/network-scripts/ifcfg-ib0
...
ONBOOT=yes
BOOTPROTO=static
IPADDR=192.168.2.1
NETMASK=255.255.0.0
...

Internal
Use Warewulf to add the ib0 interface files
../ifcfg-ib0
DEVICE=ib0
BOOTPROTO=static
IPADDR=%{NETDEVS::IB0::IPADDR}
NETMASK=%{NETDEVS::IB0::NETMASK}
ONBOOT=yes

Add this file to your Warewulf database and tell all our internal nodes (n02-n11) to use this file
# wwsh file import ifcfg-ib0 --path=/etc/sysconfig/network-scripts/ifcfg-ib0
# wwsh provision set n[02-11] --fileadd ifcfg-ib0

Set the netmask on the ib0 interface on each Warewulf node
# wwsh node set n[02-11] -D ib0 -M 255.255.0.0

Set the IP address on the ib0 interface
# for i in `seq 2 9`; do
> wwsh node set n0${i} -D ib0 -I 192.168.2.${i}
> done

# for i in `seq 10 11`; do
> wwsh node set n${i} -D ib0 -I 192.168.2.${i}
> done


OpenMPI
Allow as much pinned memory as desired (MPI needs to pin memory)
/etc/security/limits.conf
...
soft memlock unlimited
hard memlock unlimited
...


Add to your .bashrc to load the openmp-x86_64 library on shell start for all users desired
...
module load openmpi-x86_64
...

Head (Assuming you are using no scheduler and want the same workers per node all the time)
Add hosts and core counts to /etc/openmpi-x86_64/openmpi-default-hostfile
...
ibn1 slots=32
ibn2 slots=24
ibn3 slots=24
ibn4 slots=24
ibn5 slots=24
ibn6 slots=24
ibn7 slots=24
ibn8 slots=24
ibn9 slots=24
ibn10 slots=24
ibn11 slots=24

Where ibnN are forward lookups in our named configuration and slots in the number of cores per node (found with nproc)

You will also need an ssh key pair and correctly configured authorized_keys and known_hosts in ~/.ssh on each node (see ssh page)

You will also need to allow all traffic on your internal network via iptables since MPI does not use a standard set of ports.

Testing
A simple hello world program in C is used for testing.
This program needs to be accessible to all nodes so you should have it in a "shared" (NFS mounted) directory.
We made a directory /home/share where our code goes since /home is mounted on boot with NFS on all internal nodes.

#include <mpi.h>
#include <stdio.h>

int main(int argc, char** argv) {
    int id, np;
    char processor_name[MPI_MAX_PROCESSOR_NAME];
    int processor_name_len;

    MPI_Init(NULL, NULL);

    MPI_Comm_size(MPI_COMM_WORLD, &np);
    MPI_Comm_rank(MPI_COMM_WORLD, &id);
    MPI_Get_processor_name(processor_name, &processor_name_len);

    printf("Hello world from process %03d out of %03d, processor name %s\n", id, np, processor_name);

    MPI_Finalize();
    return 0;
}

Compile
# mpicc hello_mpi.c -o hello_mpi

Run
# mpirun -np N hello_mpi

(Or if you're root and you trust this program)
# mpirun --allow-run-as-root -np N hello_mpi

Where N is the number of tasks you want to run.
