Introduction

HP integrated lights out management(iLo). iLo is a feature built into HP servers that provides a very nice and user friendly way to remotely manage your servers at a bios level remotely. 

Configuration

By default iLo username is Administrator and the password is on a label on the top of your server. You can manage usernames and passwords for other usernames by watching for the iLo menu option and pressing F8 when booting up your server. That menu will look similar to below.


After your are in the iLo configuration menu you can go to the user tab at the top and add other users and manage their permissions. In the iLo configuration menu you can also set static ip addresses but we choose to create DHCP reservations for each on our DHCP server. 
Now to access the actual iLo management interface you will open a web browser and go to the ip that you assigned the servers iLo interface and you should get a login prompt that looks similar to this where you will be able to login with the username and password that you setup or with the Administrator and password from the label on the server.

After logging on you will be on a page with a basic overview of the server status and information about the server. 

Under the virtual media menu is where you are able to set boot order and one time boot options. 


After logging on in the bottom right corner there is a menu that shows whether the server is currently on or off and then remotely press the power button.


The System information section has various information about the server hardware and its status.

There is also a remote console option that we were unable to get working likely because of the way we proxy into the lan is blocking some of the ports required for it to work.

Problems


Our servers iLo interfaces were on a private internal lan that is unaccessible from our workstations. As a work around for this problem we created a SOCKS proxy so we would have access to that internal lan. 

Starting the proxy 

ssh -D 8888 eaholm@cscnsi.nmc-probe.org -p 146
The -D argument is to specify the port the proxy will listen on and needs to be a port not used by anything else on your system. 

Configuring firefox to use your proxy

Open your Firefox preferences


Go to the Advanced side tab then the Network tab at the top and Connection settings



You should now get a page similar to this and for the SOCKS host used 127.0.0.1 and the port you specified when starting the proxy
