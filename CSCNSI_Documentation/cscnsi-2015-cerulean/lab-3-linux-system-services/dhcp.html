Introduction

Dynamic Host Configuration Protocol (DHCP) is used to automatically assign IP addresses and other network configuration to systems on the network.
DHCP is implemented with two UDP port numbers. UDP port 67 is the destination port of a server, and UDP port 68 is used by the client.

 Service name   dhcpd
 Packages           dhcp dhcpd-common
 Configure  /etc/dhcp/dhcpd.conf; /etc/sysconfig/dhcpd
 Log files  /var/log/message
 Port  UDP 67 or 68


Configuration

Server

yum install dhcp

The first thing you need to do is assign the interface that you want DHCP to hand out addresses to. 

To do this you need to modify /etc/sysconfig/dhcpd
vim /etc/sysconfig/dhcpd

Assign the interface you want such and eth0 or eth1
# Command line options here
DHCPDARGS=eth0

Now save and close that file 

Copy the sample DHCP config file to /etc/dhcp/
cp /usr/share/doc/dhcp-*version number*/dhcpd.conf.sample /etc/dhcp/dhcpd.conf

Now edit the dhcpd.conf file
vim /etc/dhcp/dhcpd.conf

The config file is well commented so I went through and set the parameters that we wanted dhcp to configure

This server is going to be the official server to the lan so we set it to be authoritative
[...] 
authoritative; 
[...]


Set subnet, ip range, dns servers, domain name, default gateway
[...]
subnet 192.168.1.0 netmask 255.255.255.0 {
    range 192.168.1.20 192.168.1.50;
    option domain-name-servers ns.cerulean.com;
    option domain-name "cerulean.com";
    option routers 192.168.1.1;
    option broadcast-address 192.168.1.255;
    default-lease-time 600;
    max-lease-time 7200;
}
[...]

Static Host Configuration to do this you will need to know the mac address of the client you want to statically assign
[...]
host node01 {
    hardware ethernet 38:EA:A7:8B:02:9C;
    fixed-address 192.168.1.2;
}
[...]

I commented out all remaining unused configuration settings before saving and exiting


Client

By default your client should be setup to use DHCP but if it is not modify the /etc/sysconfig/network-scripts/ifcfg-*interface* to make it use DHCP
vim /etc/sysconfig/network-scripts/ifcfg-*interface* 

Change the BOOTPROTO line to dhcp
DEVICE=eth1
HWADDR=D8:9D:67:2B:BA:7D
TYPE=Ethernet
UUID=d53e6a3d-1418-4bec-8cd7-6f89d21e393e
ONBOOT=yes
NM_CONTROLLED=yes
BOOTPROTO=dhcp 


PXEboot Options

To enable DHCP to give out info required for PXEbooting you need to add a few options the basic DHCP config from earlier /etc/dhcp/dhcpd.conf


These parameters enable DHCP for PXEbooting and point the clients to the PXEboot file
allow booting;
allow bootp;
filename "pxelinux.0";

The next-server parameter in static host config needs to point to your PXE server
host n02 {
    hardware ethernet 38:EA:A7:8B:00:B4;
    fixed-address 192.168.1.3;
    next-server 192.168.1.1;
}

            

Problems

Did not run into any problems with DHCP setup.
