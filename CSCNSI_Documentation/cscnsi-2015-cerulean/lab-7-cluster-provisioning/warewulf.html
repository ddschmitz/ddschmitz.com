Introduction

After configuring our entire cluster to perform disk-full network booting through the manipulation of complicated low-level procedures, services, and configuration files, we were directed to move to disk-less network booting. This meant that all our work with PXEbooting, TFTP servers, etc. was to be ignored. Instead, we would use the Warewulf service to setup and configure the net booting for us.

Disk-less network booting has many advantages (especially for cluster computing) and made perfect sense for our cluster. The Warewulf service handles most of the hard work, with only a little configuration needed on our part.

Configuration

Before we could install the Warewulf packages, we had to set up some custom repository information. We created a new yum repo on our head node as follows:

#/etc/yum.repos.d/Warewulf.repo
[warewulf-rhel-6]
name="Warewulf"
enabled=1
name=Warewulf Releases -- RHEL 6
baseurl=http://warewulf.lbl.gov/downloads/repo/rhel6/
gpgcheck=0


Warewulf had a short initialization process to go through after installing all the packages. The very first thing we did was run

> wwinit ALL


Which did a lot of initialization of the system, and creating custom configuration files for our setup.
After this, we needed to make an image directory that we would export to each booting node. This directory needed to contain basic OS packages as well as
custom configurations that we put in. To create the directory, we used the built-in warewulf script

> wwmkchroot centos-6 /var/chroots/centos

We then needed to make an actual bootstrap for the nodes. We used the wwbootstrap command and gave it our own architecture to generate the bootstrap with:

> wwboostrap `uname -r`


This set up a default OS filesystem for a Centos 6 OS installation. It gave us all of our default configurations and packages. We then moved into the directory to configure custom configuration files. A complete list of the configuration files we modified is:
  • /etc/fstab
  • /etc/resolv.conf
  • /etc/ntp.conf
  • /etc/shadow
  • /etc/passwd
  • /etc/group
  • /etc/gshadow
  • /etc/yum.repos.d/*

We also did not have yum, so we installed it in our chroot directory. We also installed Ganglia, vim and NTP. 

After our chroot was set up, we used warewulf's

> wwvnfs --chroot /var/chroots/centos centos

Which set up our vnfs image and stored it in the Warewulf database as centos. 

Our cluster nodes also had the additional Lights-Out subsystems installed in them, which meant we had additional network interfaces for each node. To both avoid the detection of this second interface when booting up and allow ourselves to still have statically-set and identified Lights-Out management configuration, we took our old Lights-Out DHCP portion of /etc/dhcp/dhcpd.conf and moved it over to /etc/warewulf/dhcpd-template.conf, which works by inserting extra static configuration code into the generated HDCP configuration. In completion, we added the following to the template:

# /etc/warewulf/hdcpd-template.conf
subnet 192.168.1.0 netmask 255.255.255.0 {
}
option domain-name-servers ns.cerulean.com;
host iLo00 {
        hardware ethernet b4:b5:2f:ed:15:46;
        fixed-address 192.168.1.101;
}

host iLo01 {
        hardware ethernet b4:b5:2f:ed:26:40;
        fixed-address 192.168.1.102;
}

host iLo02 {
        hardware ethernet b4:b5:2f:ed:17:44;
        fixed-address 192.168.1.103;
}

host iLo03 {
        hardware ethernet b4:b5:2f:ed:26:68;
        fixed-address 192.168.1.104;
}

host iLo04 {
        hardware ethernet b4:b5:2f:ed:1b:ac;
        fixed-address 192.168.1.105;
}

host iLo05 {
        hardware ethernet b4:b5:2f:ed:2a:86;
        fixed-address 192.168.1.106;
}

host iLo06 {
        hardware ethernet b4:b5:2f:ed:20:e4;

        fixed-address 192.168.1.108;

host iLo08 {

At this point, we were ready to get a list of all the nodes that would be configured and booted by Warewulf. To get this list, we used Warewulf's wwnodescan utility:

> wwnodescan --netdev=eth0 --ipaddr=192.168.1.2 --netmask=255.255.255.0 --vnfs=centos --bootstrap=`uname -r` --groups=nodes n[01-10]

We then booted up each node in our cluster one at a time. Because this utility uses DHCP requests to get the MAC addresses of the nodes it is to keep track of, we made sure to give a little extra time between booting each computer to make sure that there wouldn't be any sort of race condition.

After getting all the nodes registered, we made our bootstrap image using

> wwbootstrap `uname -r`

We also needed to update Warewulf's PXE configuration to work with our newly registered nodes. This was done easily by running

> wwsh pxe update

After all this was done, our nodes were able to boot up. We were not done setting things up -- we still had some DHCP and DNS issues to resolve -- but the nodes were bootable.

Warewulf did not provide our booted nodes with any external dns information, and so we had to provide our own custom version of /ect/resolv.conf to allow our booted nodes to still be configured to use our head node as their name server. We added the /etc/resolv.conf to our chroot file system image. As with all changes to the vfs, we needed to recompile and update the image in the object database, which is done again using the wwvnfs command:

wwvnfs --chroot /var/chroots/centos centos

We also had an initial issue with the gateway not being set up correctly on our nodes; we needed each node to be configured to use our head node as their gateway. To solve this, we used the build in node modification tool build into Warewulf's shell:

> wwsh

   > node set --groups nodes --netdev=eth0 --gateway 192.168.1.1


At this point, we had nodes that would boot up on their own and be able to use the network appropriately. The last thing we needed to do was spend time making our vnfs image have the correct configuration and have the correct packages. We packages we installed were:
  • yum
  • nfs
  • gmond/Ganglia
  • rcpbind
  • rsyslog
We used the regular yum installation tool to install yum into our chroot (using the --installroot flag) and then performed the yum installs from our image. We also added configuration files to the chroot to make sure that specific configurable services would be set up on boot on all children nodes, including
  • fstab
  • /etc/ntp.conf
  • /etc/ganglia/gmond.conf
  • /etc/rsyslog.conf

One last wwvnfs command set up our image, and we were set.


Problems Encountered

The biggest and first problem we encountered was with our configuration. The Warewulf initialization chose the wrong network interface to work on, meaning even after we had obtained node information and got our servers set up, our DHCP was not set up to handle requests on our internal network. After fixing this, we forgot to update our PXE configurations.

To fix the network interface, we edited /etc/warewulf/provision.conf and changed the network device:
 
# What is the default network device that the master will use to
# communicate with the nodes?
network device = eth0

To update PXE configurations, we ran:

wwsh pxe update


We also spent a lot of time setting up our chroot exported environment image. We forgot to chkconfig this, and install that on several occasions. The image-making process and the reboot process were both lengthy, meaning these small changes took extended periods of time.

To our disbelief, the 'filesystem initialization' that Warewulf performed when creating our image file system (with the wwmkchroot command) did not include yum. As such, we needed to use yum on our server to install yum to our chroot environment. Luckily we found the --installroot flag for the yum command fairy quickly, and were able to use that to get yum installed.
