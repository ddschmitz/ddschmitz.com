Introduction

During the netboot process, Trivial File Transfer Protocol (TFTP) is used to get the initial files used to configure the boot session. Important initial files include the boot menus, PXE configuration, and kernel images.

 Service Name
  tftp, xinetd
 Packages  xinetd, tftp-server  
 Configure /etc/xinetd.d/tftp, /var/lib/tftpboot/
 Useful Commands
  
 Port  69


Configuration

XINETD and TFTP Installation

Installation of TFTP was very straightforward for our setup.

First, install the xinetd and tftp-server packages:

yum install xinetd tftp-server

Modify the TFTP server configuration. The only field we changed here was the disabled field, which we set to no.

vim /etc/xinetd.d/tftp

The resulting file should look like this:

service tftp
{
        socket_type             = dgram
        protocol                = udp
        wait                    = yes
        user                    = root
        server                  = /usr/sbin/in.tftpd
        server_args             = -s /var/lib/tftpboot
        disable                 = no
        per_source              = 11
        cps                     = 100 2
        flags                   = IPv4
}

TFTP Server Setup

The second part of making sure our TFTP server was ready to be used to netboot was making sure the correct and necessary files were set up to be shared using it. This means moving certain files into the /var/lib/tftpboot/ directory. The files we moved into it were:
  • initrd.img
  • menu.c32
  • pxelinux.0
  • prelinux.cfg/
  • vmlinuz
But before you can do this, you must install and configure Syslinux! 

First, create the tftpboot directory:
mkdir /var/lib/tftpboot

Copy menu.c32 and pxelinux.0 (found in the Syslinux directory /usr/share/syslinux/) to your tftboot directory:

cp /usr/share/syslinux/menu.c32 /var/lib/tftpboot/menu.c32
cp /usr/share/syslinux/pxelinux.0 /var/lib/tftpboot/pxelinux.0


Now copy initrd.img and vmlinuz from the repos hosted on your webserver (/var/www/html/CentOS/6/6.6/os/x86_64/isolinux/):

cp /var/www/html/CentOS/6/6.6/os/x86_64/isolinux/initrd.img /var/lib/tftpboot/initrd.img
cp /var/www/html/CentOS/6/6.6/os/x86_64/isolinux/vmlinuz /var/lib/tftpboot/vmlinuz


Next, create a new directory pxelinux.cfg in the tftboot directory. This is your PXE server configuration directory (this is where your menus will go).

mkdir /var/lib/tftpboot/pxelinux.cfg

Menus

Make a default menu file:

vim /var/lib/tftpboot/pxelinux.cfg/default

Here's what ours looks like:

DEFAULT bootstrap
PROMPT 0
TIMEOUT 5
ONTIMEOUT bootstrap

LABEL bootlocal
MENU LABEL ^1) bootlocal
LOCALBOOT 0

LABEL bootstrap
MENU LABEL ^2) bootstrap
KERNEL vmlinuz
APPEND initrd=initrd.img ksdevice=eth0 ks=http://www.cerulean.com/Scripts/anaconda-ks.cfg devfs=nomount

ksdevice tells it which interface to use, and ks points to the location of the kickstart file.
