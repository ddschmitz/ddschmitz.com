DNS

Procedures

Head Node
First install bind using Yum

$ yum install bind

next you need to set up the /etc/named.conf

// named.conf

// Provided by Red Hat bind package to configure the ISC BIND named(8) DNS

// server as a caching only nameserver (as a localhost DNS resolver only).

// See /usr/share/doc/bind*/sample/ for example named configuration files.

options {

        listen-on port 53 { 127.0.0.1; 192.168.1.1; };

        listen-on-v6 port 53 { ::1; };

        forwarders{10.0.35.34;};

        directory       "/var/named";

        dump-file       "/var/named/data/cache_dump.db";

        statistics-file "/var/named/data/named_stats.txt";

        memstatistics-file "/var/named/data/named_mem_stats.txt";

        allow-query     { localhost;192.168.1.0/24; };

        allow-transfer{localhost; 192.168.1.2; };

        recursion yes;

};

logging {

        channel default_debug {

                file "data/named.run";

                severity dynamic;

        };

}; 

zone "." IN{

type hint;

file "named.ca";

};

zone "vermilion.com" IN {

        type master;

        file "forward.vermilion.com.zone";

        allow-update { none; };

};

zone"1.168.192.in-addr.arpa" IN{

        type master;

        file "reverse.vermilion.com.zone";

        allow-update {none;};

};

include "/etc/named.rfc1912.zones";

include "/etc/named.root.key";


Importent points
  1. setting up forwarders{10.0.35.34;};
  2. allow-query { localhost;192.168.1.0/24; };
  3. remove key encryption (lines 19-27 )
  4. add look up zone and reverse  (lines 40-49)
next set up zones in /var/named/forward.vermilion.com.zone
and/var/named/reverse.vermilion.com.zone
/var/named/forward.vermilion.com.zone

$TTL 3H

@       IN SOA  ns.vermilion.com. root.vermilion.com. (

                                        0       ;serial 

                                        1D      ; refresh

                                        1H      ; retry

                                        1W      ; expire

                                        3H )    ; minimum

        IN      NS      ns.vermilion.com.


ns      IN      A       192.168.1.1

N2      IN      A       192.168.1.2

www     IN      A       192.168.1.1

ntp     IN      A       192.168.1.1

nfs     IN      A       192.168.1.1

repo    IN      A       192.168.1.1


/var/named/reverse.vermilion.com.zone

$TTL 3H

@       IN SOA  ns.vermilion.com. root.vermilion.com. (

                                        0       ;serial 

                                        1D      ; refresh

                                        1H      ; retry

                                        1W      ; expire

                                        3H )    ; minimum

        IN      NS      ns.vermilion.com.

1       IN      PTR     ns.vermilion.com.

2       IN      PTR     N2.vermilion.com.


configure /etc/resolv.conf to what is below

search vermilion.com.

nameserver 192.168.1.1


protect /etc/resolv.conf with 

$chattr +i /etc/resolv.conf

the second to last step is to open holes in the firewall

add line to iptables

-A INPUT -p udp --dport 53 -j ACCEPT

finally 

$ chkconfig --levels 235 named on
$service named restart
