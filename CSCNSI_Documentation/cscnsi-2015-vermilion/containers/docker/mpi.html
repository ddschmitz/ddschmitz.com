6. Open MPI

On Client Container


1. Start sshd and install packages.


$ service sshd start
$ yum install openmpi openmpi-devel compat-openmpi

2. Create files and add PATH changing variables to them.


$ vi /etc/profile.d/mpipath.csh

Add:

setenv PATH $PATH:/usr/lib64/openmpi/bin/

$ vi /etc/profile.d/mpipath.sh

Add:

PATH=$PATH:/usr/lib64/openmpi/bin/

After creating these you will need to load them so just leave the container and attach to it again. 

On Host Container


1. Start sshd and install packages.


$ service sshd start
$ yum install openmpi openmpi-devel compat-openmpi

2. Edit the limits.conf file.


$ vi /etc/security/limits.conf

Add these two lines:

#ftp          hard      nproc      0
#@student     -         maxlogins  4
soft memlock unlimited       <---
hard memlock unlimited       <---
# End of file

3. Create the machinefile with correct client IP's.


$ vi /home/machinefile 

Add to this file the IP addresses of the hosts that you want MPI to its jobs on. 

4. Change PATH variables and load modules


$ export PATH=$PATH:/usr/lib64/openmpi/bin/
$ export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/usr/lib64/openmpi/lib
$ module load openmpi-x86_64

5. Run the MPI program.


We used a simple Hello World C program shown below.

#include <mpi.h>
#include <stdio.h>

int main(int argc, char** argv) {
// Initialize the MPI environment
MPI_Init(NULL, NULL);

// Get the number of processes
int world_size;
MPI_Comm_size(MPI_COMM_WORLD, &world_size);

// Get the rank of the process
int world_rank;
MPI_Comm_rank(MPI_COMM_WORLD, &world_rank);

// Get the name of the processor
char processor_name[MPI_MAX_PROCESSOR_NAME];
int name_len;
MPI_Get_processor_name(processor_name, &name_len);

// Print off a hello world message
printf("Hello world from processor %s, rank %d"
    " out of %d processors\n",
    processor_name, world_rank, world_size);
//
//
// Finalize the MPI environment.
MPI_Finalize();
}

Make sure the file is in the same directory as your machinefile.  In our case it is /home/mpi.c.

Now compile it and run it.

$ mpicc mpi.c
$ mpirun --hostfile machinefile -np 24 -allow-run-as-root ./a.out

Extras


Most of the changes we made can be committed to an image.  That way they stay permanent when 
creating new containers.  Also you can add sshd to the services.sh script so it will start on start-up 
when you create a container.

$ vi /etc/profile.d/services.sh

Add:

service sshd start
