2. Docker Hub

Docker Hub

    Docker Hub is a great utility offered by Docker that we took advantage of.  Docker Hub essentially lets you
 upload an image to your organization/account and then pull it down from there at any time.  
So what you can do is create a container, then install software to it and make the needed 
changes you want.  Then you can upload it to your hub and you or other people in your group can pull 
it down and build a container off of it.  Thus allowing you all to be working in the same container environment.




Account and Organization Setup


1. Create an account.

There isn't anything special you need to do here it's just like any other website.  

2. (Optional) Create an organization.

    If you have a group that you are working with and would like to be able to collaborate and share images, you can create an organization. 

            A. Under search you should see your name with a down arrow.  Click on the arrow.
            B. You should now see + Add Organization.  Click it.
            C. Name your organization and then create it.  

2.1. Adding colleagues to your organization.

            A. Under search you should see your name with a down arrow.  Click on the arrow.
            B. Find the organization you just created. Click it.
            C. Under where you find your organization you should see a section called Manage.  Under that is Settings.  Click it.
            D. On the right under the blue Add a new group button is a pencil.  Click it. 
            E. From here you can add new members to the organization.

 Uploading an Image


1. Create a container and make changes to it that you see fit. Or make changes to an already existing container.

root@n01.cluster:/root$ docker ps
CONTAINER ID         IMAGE         COMMAND         CREATED         STATUS           PORTS                   NAMES
6a8ed7a28f12       centos:6.6    "/bin/bash"      5 days ago      Up 5 days   0.0.0.0:8888->22/tcp     drunk_archimedes


2. Create the Image

$ docker commit -m "this is a test" -a "Darrin Schmitz" \6a8ed7a28f12 testvermilion/test:0.1


       Here we have used the docker commit command.  We've specified two flags: -m and -a.  The -m flag allows us to specify a commit 
message, much like you would with a commit on a version control system.  The -a flag allows us to specify an author for our update. 
We've also specified the container we want to create this new image from 6a8ed7a28f12 (this ID can be shown above when I showed
the docker ps command) and we've specified a target for the image: testvermilion/test:0.1.  So what this does is it sends it to the testvermilion
organization(you could substitute this for a username and send it to your own hub) and it will be named test.  Finally we have specified a tag for the image: 0.1.

You can also do this same process for updating an image.  Just change the message and the tag.

3. Pushing the image to Docker Hub.

root@n01.cluster:/root$ docker images
REPOSITORY              TAG              IMAGE ID              CREATED              VIRTUAL SIZE
testvermilion/test      0.2            31b6b8c7495a         21 minutes ago            558.5 MB

$ docker push testvermilion/test:0.1

You should now be able to go to your Hub and see the image you pushed up.  Below is an example of what I saw.  


If we run docker search testvermilion we will be able to see our test image in our hub.

root@n02.cluster:/root$ docker search testvermilion
NAME                 DESCRIPTION      STARS      OFFICIAL      AUTOMATED
testvermilion/test                          0

3. Pulling the image. (I did this on a different node so I didn't have any previous images on my system)

When pulling the image make sure you supply the correct tag.  In my example i had to use 0.2

$ docker pull testvermilion/test:0.2

root@n02.cluster:/root$ docker images
REPOSITORY            TAG         IMAGE ID         CREATED         VIRTUAL SIZE
testvermilion/test    0.2       31b6b8c7495a    17 hours ago         558.5 MB
centos                6.6       8b44529354f3    11 weeks ago         202.6 MB

$ docker run -it testvermilion/test:0.2 /bin/bash
