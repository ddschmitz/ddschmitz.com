Lab 1 - Head Node Installation

Description:
During head node installation, proper cable management and labeling is very important. Every cable must have a label at each end that specifies the source and the destination of the cable. After installing the hardware, the head node needs to be configured with an internal and external network. 

Installation:
  • Plug in lights out management cables and cable manage
  • Plug in ethernet cables and cable manage
  • Plug in infiniband cables and cable manage
  • Plug in power cables and cable manage
  • Plug in KVM cables to the head node. 
  • Label all nodes and cables
  • Add users, check networking interfaces, start ssh daemon, and disable iptables

Add users for each of the team members and add password for each user 

# useradd nick
# useradd gustavo
# useradd evan
# passwd nick
# passwd gustavo
# passwd evan

Setup networking on the HEAD node by configuring ifcfg-eth0 and ifcfg-eth1

# ifconfig
# ifup eth1

Setup ifcfg-eth1

# vi /etc/sysconfig/network-scripts/ifcfg-eth1   

DEVICE=eth1
HWADDR=D8:9D:67:2B:17:ED
TYPE=Ethernet
UUID=0b68682f-1ff5-4bfc-9457-8f47d52323ac
ONBOOT=yes
NM_CONTROLLED=yes
BOOTPROTO=dhcp
GATEWAY=10.0.52.1

Setup ifcfg-eth0

# vi /etc/sysconfig/network-scripts/ifcfg-eth0
   
   
DEVICE=eth0
HWADDR=D8:9D:67:2B:17:EC
TYPE=Ethernet
UUID=b7d50096-17fd-4a73-95eb-86d9db7d6a56
ONBOOT=yes
NM_CONTROLLED=yes
BOOTPROTO=static
IPADDR=192.168.0.1
GATEWAY=10.0.52.148
NETMASK=255.255.0.0
    
Setup yum repositories to work from http://repo.newmexicoconsortium.org by configuring CentOS-Base.repo

# vi /etc/yum.repos.d/CentOS-Base.repo
         
# CentOS-Base.repo
#
# The mirror system uses the connecting IP address of the client and the
# update status of each mirror to pick mirrors that are updated to and
# geographically close to the client.  You should use this for CentOS updates
# unless you are manually picking other mirrors.
#
# If the mirrorlist= does not work for you, as a fall back you can try the
# remarked out baseurl= line instead.
#
#

[base]
name=CentOS-$releasever - Base
baseurl=http://repo.newmexicoconsortium.org/centos/$releasever/os/$basearch
gpgcheck=0
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-CentOS-6

#released updates
[updates]
name=CentOS-$releasever - Updates
baseurl=http://repo.newmexicoconsortium.org/centos/$releasever/updates/$basearch
gpgcheck=0
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-CentOS-6

#additional packages that may be useful

Update software

# yum update

Install any other software you would like to install

# yum install emacs man

Disable SELinux

# setenforce 0

Make sure SELinux is disabled on boot

Navigate to the following file

# vi /etc/selinux/config

Disable SELINUX in this file

# This file controls the state of SELinux on the system.
# SELINUX= can take one of these three values:
#     enforcing - SELinux security policy is enforced.
#     permissive - SELinux prints warnings instead of enforcing.
#     disabled - No SELinux policy is loaded.
SELINUX=disabled
# SELINUXTYPE= can take one of these two values:
#     targeted - Targeted processes are protected,
#     mls - Multi Level Security protection.
SELINUXTYPE=targeted
