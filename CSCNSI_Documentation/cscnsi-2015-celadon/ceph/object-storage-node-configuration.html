5. Object Storage Node (OSD) Configuration

Description: The Object storage daemons communicate with the proxy nodes, monitor node and the JBODS in order to GET/SET objects on the disks.

See below for details on setting up LSI Logic / Symbios Logic MegaRAID SAS 2208

Installation:
# yum install ceph
# chkconfig ceph on

Configuration:
# scp 192.168.0.2:/etc/ceph/ceph.* /etc/ceph/.
# ceph osd create

Ensure that an osd was created:
# ceph osd ls

Use the osd number listed from the previous command for {osd-number} in the following command:
# mkdir /var/lib/ceph/osd/ceph-{osd-number}

Find the drive we are using with:
# ceph-disk list

Format the drive:
# mkfs -t xfs -f /dev/{drive}

Note: If mkfs fails run the following command and then try formatting the drive and mounting it again:
# ceph-disk zap /dev/{drive}

# mount /dev/sda /var/lib/ceph/osd/ceph-{osd-number}

Initialize the OSD data directory:
# ceph-osd -i {osd-number} --mkfs --mkkey

Register the OSD authentication key:
# ceph auth add osd.{osd-number} osd 'allow *' mon 'allow rwx' -i /var/lib/ceph/osd/ceph-{osd-number}/keyring


Add the node to the CRUSH map:
# ceph osd crush add-bucket {hostname} host
# ceph osd crush move {hostname} root=default
# ceph osd crush add osd.{osd-number} 1.0 host={hostname}

Add the osd to /etc/ceph/ceph.conf:
# vi /etc/ceph/ceph.conf
...
[osd.{osd-number}]
    osd host = {hostname}
...

Update /etc/ceph/ceph.conf on the monitor node:
# scp /etc/ceph/ceph.conf {ip-address-of-monitor-node}:/etc/ceph/ceph.conf
# ssh {ip-address-of-monitor-node}

On the monitor node:
# service ceph restart
# exit

Back on the OSD node:
# touch /var/lib/ceph/osd/ceph-{osd-number}/sysvinit
# /etc/init.d/ceph start osd.{osd-number}

Check that the OSD is up:
# ceph -w

The following MAIN SCRIPT can be used to set up the required osd's on a given node:
#!/bin/bash                                                                                                   # Copy of the configuration files from the monitro node to the osd node
# This script assumes you know how many dev's there are on the system.  Please edit the range for the
# for loop if the system has a different range of dev's.
# The script also assumes /home/share/ceph.conf exists and is populated properly.
# Todo: Overautomate by using bash to automagically find the range of devices
yum install -y ceph
scp 192.168.0.2:/etc/ceph/ceph.conf /etc/ceph/.
scp 192.168.0.2:/etc/ceph/ceph.client.admin.keyring /etc/ceph/.

chk_bucket=`ceph osd tree | grep $HOSTNAME | awk '{print $4}'`

for i in {}; do
    uuid=`uuidgen`
    dev=/dev/sd$i
# Create the osd
    osd_num=`ceph osd create $uuid`

# Create the OSD data directory
    mkdir /var/lib/ceph/osd/ceph-$osd_num

# Format the drive
#    mkfs -t ext4 -E lazy_itable_init $dev
    mkfs.ext4 -F -E lazy_itable_init $dev

# Mount the drive
    mount -o user_xattr $dev /var/lib/ceph/osd/ceph-$osd_num

# Ensure that the drive is mounted on boot
    echo "mount -o user_xattr $dev /var/lib/ceph/osd/ceph-$osd_num" >> /etc/rc.local

# Initialize the OSD data directory
    ceph-osd -i $osd_num --mkfs --mkkey --osd-uuid $uuid

# Register the OSD authentication key
    ceph auth add osd.$osd_num osd 'allow *' mon 'allow profile osd' -i /var/lib/ceph/osd/ceph-$osd_num/keyring

# Create the bucket once for this OSD if it doesn't already exist
    if [ "$chk_bucket" != "$HOSTNAME" ]
    then
        echo "Creating bucket: $HOSTNAME"
        ceph osd crush add-bucket $HOSTNAME host
        ceph osd crush move $HOSTNAME root=default
        chk_bucket="$HOSTNAME"
    fi
# Add the node to the CRUSH map
    ceph osd crush add osd.$osd_num 1.0 host=`hostname`

# Ensure that the osd is mounted on boot
#    echo "$dev    /var/lib/ceph/osd/ceph-$osd_num    ext4    defaults    0 0" >> /etc/fstab

# Start up the Object Storage Daemon (OSD)
    touch /var/lib/ceph/osd/ceph-$osd_num/sysvinit
    /etc/init.d/ceph start osd.$osd_num

# Ensure that each osd started on boot
    echo "/etc/init.d/ceph start osd.$osd_num" >> /etc/rc.local
done
                                                                   



Setting up LSI Logic / Symbios Logic MegaRAID SAS 2208

On your admin node or head node make sure clustershell is installed:
# yum install clush

Edit /etc/clustershell/groups
vi # /etc/clustershell/groups
...
osds: HOST1 HOST2 ... HOSTn
...


Ensure that MegaCli is installed.  You can download is from here:
http://www.lsi.com/downloads/Public/RAID%20Controllers/RAID%20Controllers%20Common%20Files/8.07.14_MegaCLI.zip

For a more recent version search for MegaCLI in the www.lsi.com search bar.

Copy this zip file over to your admin node and place it in a file share that can be accessed by all your nodes in the cluster.

Unzip the file on the admin node and install MegaCli:
#clush -g osds rpm -i /home/share/Linux/MegaCli-8.07.14-1.noarch.rpm
#clush -g osds cp /opt/MegaRAID/MegaCli/MegaCli64 /usr/bin/.

Use MegaCli to delete all existing logical volumes:

# clush -g osds MegaCli64 -CfgLdDel -L0 -a0
Determine the number of slots and the enclosure id one node at a time using the following command:
# MegaCli64 PDList -aAll

Once you know the enclosure id and slot number edit the following scripts where we have -PhysDrv[{enclosure id}:{slot number}]:
# vi make_good.sh
#!/bin/bash
MegaCli64 -LDInfo -Lall -a0
MegaCli64 -CfgLdDel -L0 -a0
MegaCli64 AdpGetProp enablejbod -aAll
MegaCli64 AdpSetProp EnableJBOD 1 -aAll

for i in `seq 0 20`;
do
        MegaCli64 PDMakeGood -PhysDrv[93:$i] -Force -a0
        MegaCli64 PDMakeJBOD -PhysDrv[93:$i] -a0
done

Run this script on each node and modify the {enclosure id} and {slot number} accordingly.

Once this is done you can run the MAIN SCRIPT given above on each of the nodes.

NOTE: Before running the main script run:
# lsscsi

You should see a bunch of devices listed and adjust the script accordingly -- each node may not contain the same number of drives.

NOTE: If the drives do not show up after using lsscsi reboot the JBODs.
