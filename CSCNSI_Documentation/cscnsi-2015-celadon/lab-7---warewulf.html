Lab 7 - Warewulf

Description: 

Warewulf is a scalable systems management suite originally developed to manage large high-performance Linux clusters. Focused on general scalable systems management, it includes a framework for system configuration, management, provisioning/installation, monitoring, event notification, and more via a modular plugin architecture. Install the components and features you need or leverage the existing system configurations stored within Warewulf to create custom solutions to meet your particular needs.

Important
**Make a copy of your dhcp.conf file before starting, because Warewulf will delete it**
**Use wwsh to access Warewulf**

Installation:

Step 1) Install Warewulf repos
# mkdir /home/share
# chmod 0777 /home/share
# cp /etc/ntp.conf /home/share/
# cp /etc/dhcp/dhcpd.conf /home/share/
# yum install wget
# mkdir /var/chroots
# mkdir /var/chroots/centos
$ sudo wget -o /etc/yum.repos.d/warewulf-rhel6.repo http://warewulf.lbl.gov/downloads/repo/warewulf-rhel6.repo

Step 2) Install Warewulf components via YUM
$ sudo yum install warewulf-common warewulf-provision warewulf-cluster warewulf-provision-server warewulf-vnfs tcpdump perl-DBD-MySQL mysql-server tftp tftp-server

Step 3) After you install the warewulf components run wwinit ALL. This will walk through various configuration and initialization tests.
# wwmkchroot centos-6 /var/chroots/centos
# yum install yum --installroot /var/chroots/centos
# export CHROOTDIR=/var/chroots/centos
$ sudo wwinit ALL


Step 3) In the chroot:
# chroot /var/chroots/centos
# yum install ganglia-gmond rsyslog
# chkconfig gmond on
# chkconfig rsyslogd on
# chkconfig ntpd on

Configuration:

Step 4) The next step is to go into the /etc/warewulf/provisions.conf file and change the network device to eth0 instead of eth1. This allows the child nodes to pull the images from the head node through eht0.
**Just change the network device from eth1 to eth0. Everything else remains the same**
# What is the default network device that the master will use to
# communicate with the nodes?
network device = eth0

# Which DHCP server implementation should be used?
dhcp server = isc

# What is the TFTP root directory that should be used to store the
# network boot images? By default Warewulf will try and find the
# proper directory. Just add this if it can't locate it.
#tftpdir = /var/lib/tftpboot

# Automatically generate and manage a dynamnic_host virtual file
# object in the datastore? This is useful for provisioning this
# out to nodes so they always have a current /etc/hosts file.
generate dynamic_hosts = yes

# Should we manage and overwrite the local hostfile file on this
# system? This will cause all node entries to be added
# automatically to /etc/hosts.
update hostfile = yes

# If no cluster/domain is set on a node, should we add 'localdomain'
# as the default domain
use localdomain = yes

Step 6) In the chroot:
# chroot /var/chroots/centos
# yum install ganglia-gmond rsyslog bind dhcp dhcp-common ntp nfs-utils openssh-clients
# chkconfig gmond on
# chkconfig rsyslogd on
# chkconfig nfs on
# chkconfig ntpd on
# chkconfig named on

Step 7) Now we need to add and uncomment the following line in /var/chroots/centos/etc/rsyslog.conf:

#*.* @192.168.0.1:514
IMPORTANT: DO NOT uncomment this line on the head node as /var/log/messages will fill up very quickly.

If everything is working properly you will see a number of directories in /var/log/xxx.xxx.x.x where xxx.xxx.x.x is the ip of each of the child nodes forwarding their log messages to the head node.  Each of these directories will contain a file called syslog.log with the forwarded log messages.

Step 8) We need to forward DNS files to the head node. Add the following line to /var/chroots/centos/etc/named.conf:

# vi /var/chroots/centos/etc/named.conf

forwarders { 192.168.0.1; };

Step 9) Ensure that we mount /home on our child nodes:

# echo "192.168.0.1:/home    /home nfs    defaults    0 0" >> /var/chroots/centos/etc/fstab
# chattr +i /var/chroots/centos/etc/fstab

Step 10) We need to make sure change the resolv.conf file on all our child nodes vim /var/chroots/centos/etc/resolv.conf
; generated by /sbin/dhclient-script
;search newmexicoconsortium.org. nmc-probe.org.
;nameserver 10.0.35.34
;nameserver 10.0.35.23
search teamceladon.com.
nameserver 192.168.0.1

Step 11) We need to import user information into our chroot environment.
# wwsh file import /etc/passwd
# wwsh file import /etc/group
# wwsh file import /etc/gshadow
# wwsh file import /etc/shadow

Step 12) Now we need to build our image.
# wwvnfs --chroot /var/chroots/centos
# wwsh dhcp update
# wwsh dhcp restart
# wwsh pxe update

Step 13) The next step is to run a node scan.
wwnodescan --netdev=eth0 --ipaddr=192.168.0.2 --netmask=255.255.0.0 --vnfs=centos --bootstrap=`uname -r` --groups=newnodes NODE-[01-10]

**Where xxx.xxx.x.x are the ip's of the iLO for all the child nodes.**

Step 14) The final step is to power on each of the child nodes one-by-one staggering the boot of each node by about 15-30 seconds:
ipmitool -H xxx.xxx.x.x -v -I lanplus -U Celadon chassis power on

IMPORTANT:  If the nodescan has finished and you cannot ssh into all of the client nodes then you may need to run the following:
# wwsh pxe update
# wwinit all

**Don't worry if vnfs fails just reboot the nodes now.**

Installing clustershell:

# yum install clustershell
# vim /etc/clustershell/groups
